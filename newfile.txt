  git add [参数] <路径>　作用就是将我们需要提交的代码从工作区添加到暂存区，就是告诉git系统，我们要提交哪些文件，之后就可以使用git commit命令进行提交了。
  git commit 主要是将暂存区里的改动给提交到本地的版本库。每次使用git commit 命令我们都会在本地版本库生成一个40位的哈希值，这个哈希值也叫commit-id。
在使用git commit命令将修改从暂存区提交到本地版本库后，只剩下最后一步将本地版本库的分支推送到远程服务器上对应的分支，就是用git push。
  显而易见的是这三个是由顺序先后的，必须先将修改提交到暂存区，再从暂存区提交到本地库，再从本地库提交到远程服务上的库，虽然都是提交修改，但是这就是最根本的区别，修改的库不相同。
